02598   /* \brief Post propagator for multi-dimensional bin packing
02599    *
02600    * In the following \a n refers to the number of items and \a m
02601    * refers to the number of bins.
02602    *
02603    * The multi-dimensional bin-packing constraint enforces that
02604    * all items are packed into bins
02605    * \f$b_i\in\{0,\ldots,m-1\}\f$ for \f$0\leq i<n\f$
02606    * and that the load of each bin corresponds to the items
02607    * packed into it for each dimension \f$l_{j\cdot
02608    * d + k} = \sum_{\{i\in\{0,\ldots,n-1\}|
02609    * b_{j\cdot d+k}=i}\}s_{i\cdot d+k}\f$
02610    * for \f$0\leq j<m\f$, \f$0\leq k<d\f$
02611    * Furthermore, the load variables must satisfy the capacity
02612    * constraints \f$l_{j\cdot d + k} \leq
02613    * c_k\f$ for \f$0\leq j<m\f$, \f$0\leq k<d\f$.
02614    *
02615    * The constraint is implemented by the decomposition
02616    * introduced in: Stefano Gualandi and Michele Lombardi. A
02617    * simple and effective decomposition for the multidimensional
02618    * binpacking constraint. CP 2013, pages 356--364.
02619    *
02620    * Posting the constraint returns a maximal set containing conflicting
02621    * items that require pairwise different bins.
02622    *
02623    * Note that posting the constraint has exponential complexity in the
02624    * number of items due to the Bron-Kerbosch algorithm used for finding
02625    * the maximal conflict item sets.
02626    *
02627    * Throws the following exceptions:
02628    *  - Of type Int::ArgumentSizeMismatch if any of the following properties
02629    *    is violated: \f$|b|=n\f$, \f$|l|=m\cdot d\f$, \f$|s|=n\cdot d\f$,
02630    *    and \f$|c|=d\f$.
02631    *  - Of type Int::ArgumentSame if \a l and \a b share unassigned variables.
02632    *  - Of type Int::OutOfLimits if \a s or \a c contains a negative number.
02633    *
02634    * \ingroup TaskModelIntBinPacking
02635    */
