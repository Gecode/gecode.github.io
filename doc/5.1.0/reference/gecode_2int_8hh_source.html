02693   /* \brief Post propagator for multi-dimensional bin packing
02694    *
02695    * In the following \a n refers to the number of items and \a m
02696    * refers to the number of bins.
02697    *
02698    * The multi-dimensional bin-packing constraint enforces that
02699    * all items are packed into bins
02700    * \f$b_i\in\{0,\ldots,m-1\}\f$ for \f$0\leq i<n\f$
02701    * and that the load of each bin corresponds to the items
02702    * packed into it for each dimension \f$l_{j\cdot
02703    * d + k} = \sum_{\{i\in\{0,\ldots,n-1\}|
02704    * b_{j\cdot d+k}=i}\}s_{i\cdot d+k}\f$
02705    * for \f$0\leq j<m\f$, \f$0\leq k<d\f$
02706    * Furthermore, the load variables must satisfy the capacity
02707    * constraints \f$l_{j\cdot d + k} \leq
02708    * c_k\f$ for \f$0\leq j<m\f$, \f$0\leq k<d\f$.
02709    *
02710    * The constraint is implemented by the decomposition
02711    * introduced in: Stefano Gualandi and Michele Lombardi. A
02712    * simple and effective decomposition for the multidimensional
02713    * binpacking constraint. CP 2013, pages 356--364.
02714    *
02715    * Posting the constraint returns a maximal set containing conflicting
02716    * items that require pairwise different bins.
02717    *
02718    * Note that posting the constraint has exponential complexity in the
02719    * number of items due to the Bron-Kerbosch algorithm used for finding
02720    * the maximal conflict item sets.
02721    *
02722    * Throws the following exceptions:
02723    *  - Of type Int::ArgumentSizeMismatch if any of the following properties
02724    *    is violated: \f$|b|=n\f$, \f$|l|=m\cdot d\f$, \f$|s|=n\cdot d\f$,
02725    *    and \f$|c|=d\f$.
02726    *  - Of type Int::ArgumentSame if \a l and \a b share unassigned variables.
02727    *  - Of type Int::OutOfLimits if \a s or \a c contains a negative number.
02728    *
02729    * \ingroup TaskModelIntBinPacking
02730    */

04991   /*
04992    * \brief Relaxed assignment of variables in \a x from values in \a sx
04993    *
04994    * The variables in \a x are assigned values from the assigned variables
04995    * in the solution \a sx with a relaxation probability \a p. That is,
04996    * if \$fp=0.1\f$ approximately 10% of the variables in \a x will be
04997    * assigned a value from \a sx.
04998    *
04999    * The random numbers are generated from the generator \a r. At least
05000    * one variable will not be assigned: in case the relaxation attempt
05001    * would suggest that all variables should be assigned, a single
05002    * variable will be selected randomly to remain unassigned.
05003    *
05004    * Throws an exception of type Int::ArgumentSizeMismatch, if \a x and
05005    * \a sx are of different size.
05006    *
05007    * Throws an exception of type Int::OutOfLimits, if \a p is not between
05008    * \a 0.0 and \a 1.0.
05009    *
05010    * \ingroup TaskModelInt
05011    */
