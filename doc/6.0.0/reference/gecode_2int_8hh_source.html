02820              const IntVarArgs& l,
02821              const IntVarArgs& b, const IntArgs& s,
02822              IntPropLevel ipl=IPL_DEF);
02823   /* \brief Post propagator for multi-dimensional bin packing
02824    *
02825    * In the following \a n refers to the number of items and \a m
02826    * refers to the number of bins.
02827    *
02828    * The multi-dimensional bin-packing constraint enforces that
02829    * all items are packed into bins
02830    * \f$b_i\in\{0,\ldots,m-1\}\f$ for \f$0\leq i<n\f$
02831    * and that the load of each bin corresponds to the items
02832    * packed into it for each dimension \f$l_{j\cdot
02833    * d + k} = \sum_{\{i\in\{0,\ldots,n-1\}|
02834    * b_{j\cdot d+k}=i}\}s_{i\cdot d+k}\f$
02835    * for \f$0\leq j<m\f$, \f$0\leq k<d\f$
02836    * Furthermore, the load variables must satisfy the capacity
02837    * constraints \f$l_{j\cdot d + k} \leq
02838    * c_k\f$ for \f$0\leq j<m\f$, \f$0\leq k<d\f$.
02839    *
02840    * The constraint is implemented by the decomposition
02841    * introduced in: Stefano Gualandi and Michele Lombardi. A
02842    * simple and effective decomposition for the multidimensional
02843    * binpacking constraint. CP 2013, pages 356--364.
02844    *
02845    * Posting the constraint returns a maximal set containing conflicting
02846    * items that require pairwise different bins.
02847    *
02848    * Note that posting the constraint has exponential complexity in the
02849    * number of items due to the Bron-Kerbosch algorithm used for finding
02850    * the maximal conflict item sets.
02851    *
02852    * Throws the following exceptions:
02853    *  - Of type Int::ArgumentSizeMismatch if any of the following properties
02854    *    is violated: \f$|b|=n\f$, \f$|l|=m\cdot d\f$, \f$|s|=n\cdot d\f$,
02855    *    and \f$|c|=d\f$.
02856    *  - Of type Int::ArgumentSame if \a l and \a b share unassigned variables.
02857    *  - Of type Int::OutOfLimits if \a s or \a c contains a negative number.
02858    *

05140    * \brief Relaxed assignment of variables in \a x from values in \a sx
05141    *
05142    * The variables in \a x are assigned values from the assigned variables
05143    * in the solution \a sx with a relaxation probability \a p. That is,
05144    * if \$fp=0.1\f$ approximately 10% of the variables in \a x will be
05145    * assigned a value from \a sx.
05146    *
05147    * The random numbers are generated from the generator \a r. At least
05148    * one variable will not be assigned: in case the relaxation attempt
05149    * would suggest that all variables should be assigned, a single
05150    * variable will be selected randomly to remain unassigned.
05151    *
05152    * Throws an exception of type Int::ArgumentSizeMismatch, if \a x and
05153    * \a sx are of different size.
05154    *
05155    * Throws an exception of type Int::OutOfLimits, if \a p is not between
05156    * \a 0.0 and \a 1.0.
05157    *
05158    * \ingroup TaskModelInt
05159    */
