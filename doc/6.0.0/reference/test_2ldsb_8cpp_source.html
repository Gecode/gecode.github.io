01193   /* This test should fail if the recomputation-handling does not work
01194    * properly.
01195    *
01196    * Why recomputation can be a problem
01197    * ==================================
01198    *
01199    * Every branch point in LDSB is binary, with a left and a right
01200    * branch.  Whenever backtracking happens -- when a right branch is
01201    * explored -- LDSB computes a set of symmetric literals to
01202    * exclude.
01203    *
01204    * !!! This calculation may depend on the current domains of the
01205    * !!! variables.
01206    *
01207    * During recomputation, parts of the search tree are replayed.  To
01208    * be specific, the branching constraints are posted, but no
01209    * propagation happens.  This means that at a given branch point,
01210    * the domains during recomputation may be different (weaker) than
01211    * they were the first time during search.
01212    *
01213    * !!! This *cannot* cause solutions to be missed --- LDSB will not
01214    * !!! be incorrect --- but it *does* change what will be pruned.
01215    *
01216    * If recomputation is not handled properly, the difference in
01217    * domains will cause extra solutions to be found.  This is a result
01218    * of symmetries failing to be broken.
01219    *
01220    */
01221
