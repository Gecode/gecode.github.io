<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>I am attempting to implement an edge-finding algorithm that
is similar to that of firstsDist used in Mozart.&nbsp; In order to do this, two
major events should occur for every branch:<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>- Propagation:&nbsp; Generate propagators that specify that
a given task will be first.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>- Accounting:&nbsp; &nbsp;Keep track of what tasks have been
ordered in some kind of data structure.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'>I am assuming that a left branch specifies that a given task
should be first while a right branch specifies that the same task is not
first.&nbsp; In the case that a task is not first, the appropriate data
structure will need to be updated so that the task should not be chosen until
another task goes first.&nbsp; It appears that streams are used to accomplish
this type of accounting in Mozart.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;
font-family:Arial'><o:p>&nbsp;</o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;

this point the Space instance cannot be modified, but the Branching instance
can be modified in order to keep track of the tasks.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;

be ordered first. The Space instance is still not modifiable here.<o:p></o:p></span></font></p>

<p class=MsoNormal><font size=2 face=Arial><span style='font-size:10.0pt;

the Branching instance should not be modified.&nbsp; Is this a correct
assumption?&nbsp; Depending on the branching alternative, I need to specify
that the chosen task is first or specify that the chosen task cannot be first
using a data structure for accounting purposes.<o:p></o:p></span></font></p>
