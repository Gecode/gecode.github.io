<p class=MsoNormal style='margin-bottom:12.0pt'>Juan Carlos wrote:<o:p></o:p></p>

<p class=MsoNormal>Now I see, I was confused about how the backtrack works.<br>
<br>
The idea is a Local Search Method, &nbsp;first find a basic solution, and then
make little changes &nbsp;till the best solution is found, here are the steps:<br>
<br>
1)choose the best value for each variable (this value is selected by a function
based on external information), once &nbsp;all the variables had been assigned
thats a basic solution.<br>
2) Based on that solution , re-assign some variables in order to find a better
solution in a fast way. The variables that are going to &nbsp;be re-inserted
are chosen by a function based on external information and the value &nbsp;is
chosen by the same function of step 1.<br>
3) repeat 2 till no solution left.<br>
<br>
By external information I mean the information of the problem instance.<br>
<br>
Now I have some ideas about how to implement this local search but they are not
clear yet. Any suggestions are welcome.<o:p></o:p></p>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

</div>

<p class=MsoNormal style='margin-bottom:12.0pt'>That sounds a lot like large neighborhood
search. &nbsp;I'd try using a plain BAB search to find a solution, and then
start a nested search, where you create a new space, assign some of the
variables, and run some limited BAB (e.g. using a time limit). &nbsp;Once you
find a better solution there, you reuse that for the original,
&quot;master&quot; BAB search. If you don't find a better solution within the
time limit, just go on in the master search.<br>
To implement this, you'll probably have to implement your own search engines or
at least modifiy ours, but it should be possible.<br>
