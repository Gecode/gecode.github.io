<p class=MsoNormal style='margin-bottom:12.0pt'>Malcolm Ryan wrote:<br>
<br>
&gt; I'd like to express the constraint:<br>
&gt;<br>
&gt; A = (B - C) mod n<br>
&gt; 0 &lt;= A &lt; n<br>
&gt;<br>
&gt; for variables A,B,C and constant n.<br>
&gt;<br>
&gt; How would you advise doing this? The obvious answer is to add an extra<br>
&gt; variable X:<br>
&gt;<br>
&gt; A = (B-C) + n * X<br>
&gt; 0 &lt;= A &lt; n<br>
&gt;<br>
&gt; but I'm not sure if this is best. Can you advise?<o:p></o:p></p>

<p class=MsoNormal>As there's no built-in propagator for the modulo constraint,
you have<br>
to decompose into other constraints, so your solution should be ok.<br>
After all, it's just a linear equation with constant coefficients<br>
1*A-1*B+1*C-n*X=0, right? &nbsp;And X will be determined as soon as A,B,C<br>
are, you don't even have to include X in the branching.<br>
