<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>That looks indeed easy enough. However I would like to have if
you guys would integrate it into the gecode/support part as it is really
useful.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>Then, in 3.0.0 there will be three memory areas for allocation:<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'> - The global heap (there a normal stl allocator works anyway).<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'> - Space allocated memory: so one would just have to adapt to
the new Gecode 3.0.0 memory management.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>- Region allocated memory for temporary memory usage. That's new
in 3.0.0.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>Regions work as follows: if one has a space one can get a region
for that space. One can allocate memory (very very efficiently) from a region.
However, the memory will only be freed when the region is destructed.
(Actually, what happens there is that all spaces for one thread share a small
memory area and that a Region provides access to this shared memory area. If a
request cannot be fulfilled from this area, memory will be allocated
transparently from the heap). Very simple as you don't have to deallocate to a
region, very efficient, and much cleaner than alloca.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'><o:p>&nbsp;</o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'>So, what I would like to ask you guys: would you be willing to<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'> - Write stl allocators for both regions and spaces.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'> - Test with your own programs that these allocators work.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'> - Send me a patch for Gecode 3.0.0 so that I can integrate your
stuff into the trunk into gecode/support (as a separate file).<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'> - Test the stuff again after I integrated.<o:p></o:p></span></p>

<p class=MsoNormal><span style='font-size:11.0pt;font-family:"Calibri","sans-serif";
color:#1F497D'> - Negotiate among you who does what.<o:p></o:p></span></p>
