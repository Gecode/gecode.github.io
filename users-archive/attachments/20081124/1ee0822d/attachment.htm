</div>

<div>

<p class=MsoNormal>2. Post some constraints on the derived ordered start times<o:p></o:p></p>

</div>

<div>

<p class=MsoNormal><o:p>&nbsp;</o:p></p>

</div>

<div>

<p class=MsoNormal>Here is the example code...<br>
<br>
IntVarArray start;<br>
IntVarArray end;<br>
IntVarArray startOrder;<br>
IntVarArray order;<br>
<br>
.....................<br>
.....................<br>
.....................<br>
// start and end constraints are well defined<br>
.....................<br>
.....................<br>
for( int i=0; i&lt;nTasks; i++) {<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; post( this, start, order[i], startOrder );<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; post( this, end, order[i], endTimeStamps );<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; if( i &lt; nTasks-1 )<br>
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; post( this,
startOrder[i] &lt; startOrder[i+1] );<br>
}<br>
<br>
// Adding some post constraints on the ordered tasks<br>
for( int i=0; i&lt;nTasks-1; i++ ) {<br>
&nbsp;&nbsp;&nbsp;&nbsp; post( this, endTimeStamps[i] + switchingCost[i+1]&nbsp;
&lt; startOrder[i+1] );<br>
}<br>
<br>
If we add the above constraint, it is not providing any solution. Instead if we
give the same constraints with predefined input order values like given below,
it is providing solution.<br>
<br>
int order[] = { 0, 1, 2, 3, 4, 5 };<br>
......................<br>
for( int i=0; i&lt;nTasks-1; i++ ) {<br>
&nbsp;&nbsp;&nbsp;&nbsp; post( this, endTimeStamps[i] +
