<p class=MsoNormal>Hi,<br>
<br>
Currently it is not possible to exchange the random number generator.<br>
Furthermore, the implementation uses properties of the random number<br>
generator that would most probably compromise efficiency if they were<br>
abstracted away (for example, the state is limited to a single<br>
unsigned int as seed). If you want to use your own random number<br>
generator, I would suggest writing your own subclasses of ViewSelBase<br>
and ValSelbase similar to ViewSelRnd and ValRnd.<br>
<br>
For curiosity, what is the reason that you want to change the random<br>
number generator used?<br>
<br>
Cheers,<br>
Mikael<o:p></o:p></p>
