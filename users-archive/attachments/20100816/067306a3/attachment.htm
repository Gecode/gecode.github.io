<div>1) a_i returns ES_FAILURE. My guess is that the propagator wont be run, as the entire space will become failed. Am I right?</div></blockquote><div><br></div><div>When an advisor returns ES_FAILURE, the current tell will report failure. The propagator that made the tell will then see that and report failure back to the kernel, and propagation will be aborted.</div>


<div><br></div><div><br></div><blockquote class="gmail_quote" style="margin:0 0 0 .8ex;border-left:1px #ccc solid;padding-left:1ex"><div>2) a_i returns ES_NOFIX. My guess here is that the propagator will  be scheduled, how ever what happen if after this advisor, the advisor a_i+1 returns ES_FIX?. IMO the right thing to schedule the propagator, but is this done by gecode automatically?</div>


</blockquote><div><br></div><div>If a propagator has been scheduled for execution, it will be executed when the propagation queue reaches it. Returning ES_FIX will not affect the scheduled status of the advisors propagator in any way.</div>
