Let me be the second user to complain, then.  I built two very similar applications against Gecode 3.4.2 (VS 2010 and Win 7).  One worked fine, and the other couldn&#39;t find the C runtime.  We compared the project settings between the two, and found that the first was actually picking up the C runtime from another library in the project just by luck.  Stealing the manifest from the working binary and adding it to the second binary caused the second to work as well.  Obviously that&#39;s not the ideal solution long term, and really only worked because the dependencies were fairly simple in both cases.<br>
