The problem is following. <br>
I have a set of objects with some properties of types such as string, double, integer, etc.<br>

The input for my program is an object that has dependencies on the objects from the domain. Apart from that the input object puts constraints on the properties of the objects from object domain. Here's an example:<br>
<br>
--------------------------------------<br>
InputObject O1<br>
&nbsp;&nbsp;&nbsp; property1 = &quot;name&quot;<br>
&nbsp;&nbsp;&nbsp; property2 = 4.321<br>
&nbsp;&nbsp;&nbsp; property3 = 2<br>
&nbsp;&nbsp;&nbsp; <br>
&nbsp;&nbsp;&nbsp; dependency1 = &quot;objectName&quot;<br>
--------------------------------------<br>
<br>
Space:<br>
D1 {obj1, obj2, obj3, ...}<br>
<br>
The idea is to select one or more objects from D1 such that the name of an object is &quot;objectName&quot; AND it satisfies the constraints (property1, property2, etc.). There can be dozens of objects with &quot;objectName&quot;, but only few can satisfy the constraints. And
 this chain can be infinite, i.e. if selected object is obj1, it can also have dependencies on others (i.e. obj23), and it goes on and on.<br>
