    Hi,<br>
    <br>
    I think that the element constraint does not exactly encode your
    purpose. You write:<br>
    <br>
    y ∈ g1_reach[x] and z ∈ g1_reach[y] ==>  z ∈ g1_reach[x]<br>
    <br>
    while your element constraint represents something like (if I am
    correct)<br>
    <br>
    y ∈ g1_reach[x] and z ∈ g1_reach[y] <=> z ∈ g1_reach[x]<br>
    <br>
    i.e., this is an equivalence.<br>
    <br>
    I do not know why you would have thousands of solution, though.
    Neither why they do not respect the "superset" constraint.<br>
    <br>

    <blockquote cite="mid:52CFBEED.5010002@pankrath.net" type="cite">Hello,
      <br>
      <br>
      I'm quite new to constraint programming and gecode in particular
      and ran into some issue
      <br>
      <br>
      I have to directed graphs G1=(V1,V1×V1), G2=(V2, V2×V2) and I am
      searching for a mapping h: V1 → V2, that mantains reachability.
      I.e. if there is a path v1 -*> v2 (v1,v2 ∈ V1) than there must
      also be a path h(v1) -*> h(v2).
      <br>
      <br>
      To model this using gecode, I am using 4 SetVarArrays:
      <br>
      <br>
      SetVarArray g1_edges; // y ∈ g1_edges[x]: there is an edge from
      v_x to v_y in G1
      <br>
      SetVarArray g2_edges; // like above for G2
      <br>
      <br>
      SetVarArray g1_reach; // y ∈ g1_reach[x]: there is a path from v_x
      to v_y in G1
      <br>
      SetVarArray g2_reach; // like above for G2.
      <br>
      <br>
      Now I want gecode to calculate the reachability of those graphs.
      Thus I tried to fix the graph by the following domain constraints.
      A picture of the desired graph is attached to this mail.
      <br>
      <br>
       // define edges between nodes
      <br>
       dom(*this, g1_edges[0], SRT_EQ, IntSet(1, 2));
      <br>
       dom(*this, g1_edges[1],  SRT_EQ, 5 );
      <br>
       dom(*this, g1_edges[2], SRT_EQ,  IntSet(3,4));
      <br>
       dom(*this, g1_edges[3],  SRT_EQ, 5);
      <br>
       dom(*this, g1_edges[4],  SRT_EQ, 5 );
      <br>
       dom(*this, g1_edges[5],  SRT_EQ, IntSet::empty);
      <br>
      <br>
      Then I want to encode the following relations:
      <br>
          * y ∈ g1_reach[x], if y ∈ g1_edges[x] (direct neighbours)
      <br>
          * if y ∈ g1_reach[x] and z ∈ g1_reach[y] then z ∈ g1_reach[x]
      (transitivity)
      <br>
      <br>
      Thus I do:
      <br>
      for(int x = 0; x < 6; ++x)
      <br>
      {
      <br>
          rel(*this, g1_reach[x] >= g1_edges[x]); // superset
      <br>
          element(*this, g1_reach, SOT_UNION, g1_reach[x], g1_reach[x]);
      <br>
          // call to element taken from the manual, section 5.2.4
      <br>
          // is it correct?
      <br>
      }
      <br>
      <br>
      Then I'll branch on g1_reach, like this:
      <br>
      branch(*this, g1_reach, SET_VAR_NONE(), SET_VAL_MIN_INC());
      <br>
      <br>
      However this gives me a >1000 solutions and all look like
      <br>
      g1_edges {{1,2}, {5}, {3,4}, {5}, {5}, {}}
      <br>
      g1_reach {}
      <br>
      <br>
      What is the correct way to model this?
      <br>
