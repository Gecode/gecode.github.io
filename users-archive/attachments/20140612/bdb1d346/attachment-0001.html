Hello Christian,<br>
I would like to know if there is an easy way to implement a special constraint.<br>
I have an array with 10 numbers.<br>
My constraint would be : if you plot the array values in function of the index,  you should have a wave aspect /\.<br>
For example with 10 numbers<br>
<br>
1 3 5 9 11 7 6 5 4 2 0 (increasing then decreasing) There is no importance at witch index there is the inflexion point.<br>
<br>
Would you have an idea how to implement that in a clean way?<br>
<br>
</blockquote>
<br></div></div>
For reference, this is similar to the inflexion constraint (<a href="http://www.emn.fr/z-info/sdemasse/gccat/Cinflexion.html" target="_blank">http://www.emn.fr/z-info/<u></u>sdemasse/gccat/Cinflexion.html</a><u></u>) with the number of inflexions set to 1 (assuming that valleys are also ok).<br>

<br>
The simplest way in my opinion to model this constraint would be the following.<br>
<br>
  Assume the sequence of numbers is called x and has n entries.<br>
  Define a new boolean array of variables y with n-1 values.<br>
  For each place i in y, add the constraint<br>
    y[i] = (x[i] < y[i])<br>
  (that is, reified ordering).<br>
  On the y variables, add the constraint regular(y, 1*0*)<br>
<br>
The ordering constraint and the regular expression can be modified to suit your particular needs.<br>
